# MIN-40 信息架构与核心用户流程

## 1. 目标

为 RightCleaner 定义首版可执行的信息架构与核心用户流程，覆盖以下范围：

- 首页
- 清理项列表
- 清理项详情
- 批量操作
- 备份与恢复

目标是让用户能在最短路径内完成「发现可清理内容 -> 评估风险 -> 执行清理 -> 必要时恢复」的完整闭环。

## 2. 产品原则

1. 风险前置：任何删除动作之前，都必须让用户先看到影响范围、可恢复性和预计释放空间。
2. 决策分层：把「系统推荐」「用户确认」「高风险二次确认」拆开，避免误删。
3. 恢复闭环：所有可逆清理默认进入备份链路，恢复入口不能埋得太深。
4. 批量优先：列表与详情都能汇入批量操作，不要求用户逐项重复确认。
5. 状态透明：扫描中、可清理、已清理、恢复中、恢复失败都要有明确状态反馈。

│  └─ 备份恢复入口
├─ 清理项列表
│  ├─ 分类视图
│  ├─ 筛选与排序
│  ├─ 批量选择
│  └─ 批量操作栏
├─ 清理项详情

│  ├─ 风险等级与依赖影响
│  ├─ 备份策略

## 4. 页面定义

### 4.1 首页 / 总览

**页面目标**

- 让用户快速理解当前可优化空间与风险分布。
- 提供一次扫描、继续上次任务、进入恢复的主入口。

**核心模块**

1. 顶部状态区
   - 设备/磁盘概览
   - 最近一次扫描时间
   - 当前可释放空间估算
2. 快捷操作区
   - `开始扫描`
   - `查看推荐清理`
   - `打开备份恢复`
3. 推荐清理卡片
   - 按系统垃圾、应用缓存、大文件残留、重复文件、下载目录等分组展示
   - 展示数量、体积、风险等级
4. 风险提醒区
   - 高风险项目数量
   - 需手动确认的清理类型
5. 最近操作记录
   - 最近清理任务
   - 最近恢复任务

**主要交互**

- 点击 `开始扫描` -> 进入扫描态 -> 完成后刷新首页摘要，并可跳转列表。
- 点击推荐卡片 -> 带分类筛选跳转到列表页。
- 点击最近任务 -> 进入对应任务详情或备份详情。

### 4.2 清理项列表

**页面目标**

- 承担主决策场景，让用户筛选、比较并批量选择待清理内容。

**核心模块**

1. 顶部工具栏
   - 搜索
   - 分类筛选
   - 风险等级筛选
   - 是否支持恢复筛选
   - 排序：空间占用 / 风险 / 更新时间 / 推荐度
2. 列表主体
   - 每一行展示名称、分类、命中数、预计释放空间、风险等级、是否可恢复
   - 支持勾选
   - 支持展开摘要
3. 批量操作栏
   - 已选数量
   - 预计释放空间
   - `加入批量清理`
   - `取消选择`
4. 空状态/异常状态
   - 无命中
   - 扫描失败
   - 需要重新扫描

**主要交互**

- 点击行 -> 进入详情页。
- 勾选多项 -> 底部或顶部出现批量操作栏。
- 变更筛选条件 -> 保留当前选择，但对不在当前结果集的项显示“已隐藏已选项目”提示。

### 4.3 清理项详情

**页面目标**

- 让用户在执行前理解该项的来源、风险、明细和恢复方式。

**核心模块**

1. 基本信息
   - 名称、分类、风险等级、预计释放空间
2. 命中明细
   - 文件/目录样例
   - 命中数量
   - 最近更新时间
3. 影响说明
   - 删除后影响
   - 是否影响登录态、离线数据、应用启动性能
4. 备份策略
   - 默认是否备份
   - 备份保留周期
   - 备份体积预估
5. 操作区
   - `立即清理`
   - `加入批量`
   - `返回列表`

**主要交互**

- 点击 `立即清理` -> 进入单项确认弹层。
- 点击 `加入批量` -> 加入待处理队列，并提示返回列表继续选择或直接去批量确认。

### 4.4 批量操作确认

**页面目标**

- 汇总用户已选项目，完成批量清理前的最终确认。

**核心模块**

1. 选中项汇总
   - 分组展示
   - 每项可单独移除
2. 汇总指标
   - 预计释放空间
   - 高风险项数量
   - 将创建的备份数量
3. 风险提醒
   - 高风险项二次确认
   - 不支持恢复的项目显著提示
4. 执行配置
   - 是否自动备份
   - 失败时是否继续后续项
   - 清理完成后是否展示结果明细
5. 结果页
   - 成功/失败/跳过数量
   - 实际释放空间
   - 失败原因与重试入口

**主要交互**

- 从列表批量进入。
- 从详情页连续加入后进入。
- 执行后可前往结果页，再跳转到备份中心查看可恢复项。

### 4.5 备份与恢复

**页面目标**

- 让用户能查到清理前备份，并在必要时恢复单项或整批数据。

**核心模块**

1. 备份列表
   - 按任务批次展示
   - 包含创建时间、来源任务、体积、状态、过期时间
2. 备份详情
   - 备份包含的清理项
   - 恢复影响说明
   - 冲突文件提示
3. 恢复确认
   - 恢复范围选择：整批 / 单项
   - 是否覆盖现有内容
4. 恢复结果
   - 成功/失败数量
   - 冲突项说明

**主要交互**

- 首页可直接进入。
- 清理完成结果页提供“查看备份”入口。
- 备份过期前需要有明确提示。

## 5. 核心用户流程

### 流程 A：首次使用快速清理

```text
首页
-> 开始扫描
-> 首页刷新推荐分组
-> 进入列表查看推荐项
-> 勾选低风险项目
-> 批量操作确认
-> 开启自动备份
-> 执行清理
-> 查看结果
```

**关键决策点**

- 扫描完成后默认推荐低风险、可恢复项目。
- 高风险项目不进入默认选中集合。

### 流程 B：用户从列表精细筛选后批量清理

```text
列表
-> 筛选分类/风险等级
-> 浏览并勾选多个项目
-> 查看批量栏汇总
-> 进入批量确认
-> 移除不确定项目
-> 执行清理
-> 返回结果页
```

**关键决策点**

- 用户切换筛选后，已选集合保留。
- 高风险项在确认页单独分区，避免混在低风险项中被忽视。

### 流程 C：从详情单项清理

```text
列表
-> 详情
-> 查看命中明细与影响说明
-> 立即清理
-> 单项确认
-> 执行
-> 结果页
```

**关键决策点**

- 详情页是“理解风险”的主场，不做信息折叠过深设计。
- 单项清理也要复用备份逻辑，保证一致性。

### 流程 D：清理后恢复

```text
首页 / 结果页
-> 进入备份与恢复
-> 查看备份列表
-> 打开备份详情
-> 选择整批恢复或单项恢复
-> 二次确认
-> 恢复结果
```

**关键决策点**

- 恢复前必须提示覆盖风险。
- 恢复结果需要给出失败项和原因，便于重试。

## 6. 页面跳转关系

| 来源 | 触发动作 | 目标 |
|---|---|---|
| 首页 | 开始扫描 | 首页扫描态 / 扫描完成态 |
| 首页 | 点击推荐卡片 | 列表（带筛选条件） |
| 首页 | 查看全部推荐 | 列表 |
| 首页 | 打开备份恢复 | 备份与恢复 |
| 列表 | 点击某项 | 详情 |
| 列表 | 点击批量清理 | 批量操作确认 |
| 详情 | 立即清理 | 单项确认 / 执行结果 |
| 详情 | 加入批量 | 批量操作确认 |
| 结果页 | 查看备份 | 备份与恢复 |
| 备份列表 | 点击备份 | 备份详情 |
| 备份详情 | 点击恢复 | 恢复确认 / 恢复结果 |

## 7. 状态与反馈要求

### 全局状态

- 未扫描
- 扫描中
- 扫描完成
- 清理执行中
- 清理完成
- 恢复执行中
- 恢复完成
- 失败/部分失败

### 反馈规则

1. 扫描过程要可见当前进度与已发现项数。
2. 清理前展示“预计释放空间”，清理后展示“实际释放空间”。
3. 失败必须落到具体项目级别，不能只给全局报错。
4. 任何不可恢复操作都要在按钮附近显式标识。

## 8. 权限与风险控制

1. 高风险项默认不选中。
2. 不可恢复项必须经过二次确认。
3. 批量任务中同时包含高风险与低风险项时，确认页分区展示。
4. 恢复操作若检测到文件冲突，必须在执行前提示覆盖策略。

## 9. MIN-40 交付边界

本任务交付以下内容：

- RightCleaner 顶层信息架构
- 首页、列表、详情、批量操作、备份恢复五类核心页面定义
- 四条核心用户流程
- 页面跳转矩阵
- 状态反馈与风险控制规则

本任务暂不展开：

- 视觉稿
- 高保真原型
- 技术实现细节
- 文案逐字稿

## 10. 验收标准

1. 用户可从首页在 3 步内进入推荐清理。
2. 用户可从列表完成多选并进入统一确认页。
3. 用户可从详情理解风险并执行单项清理。
4. 清理完成后，用户能从结果页或首页进入恢复链路。
5. 高风险、不可恢复、冲突恢复三类风险都有明确提示节点。
